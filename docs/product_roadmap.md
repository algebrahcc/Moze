# Moze 产品分析与发展规划 (2026.03)

## 1. 当前架构与功能审计

Moze 目前处于 **Web-First 个人财务管理系统 (PFM)** 的形态，定位于“极简、现代、资产增值视角”的记账工具。

### 1.1 导航结构合理性评估

当前 Tab 栏设计：**总览 (Dashboard)** | **资产 (Accounts)** | **收支 (Transactions)** | **报表 (Reports)** | **净值 (Investments)**

| 模块 | 定位与合理性 | 改进建议 |
| :--- | :--- | :--- |
| **总览** | **[合理]** 全局仪表盘，展示最关键指标（净资产、本月结余、近期趋势）。 | 增加“待办事项”（如未记账提醒、信用卡还款提醒）。 |
| **资产** | **[合理]** 存量管理，这是财务系统的基石。 | 增加“负债”显性展示；支持账户分组（如：日常/投资/固定资产）。 |
| **收支** | **[合理]** 流量管理，核心记账入口。 | 增加“日历视图”；支持批量操作。 |
| **报表** | **[合理]** 统计分析，回顾与决策依据。 | 增加“年度账单”生成；支持自定义时间段对比。 |
| **净值** | **[特色]** 投资视角。不仅记流水，更关注“睡后收入”与资产波动。 | 建议更名为“**财富**”或“**投资**”，涵盖投资组合、持仓分析，避免与“资产”概念混淆。 |
| **设置** | **[合理]** (已移入右上角) 低频管理操作。 | 增加数据导出/备份入口。 |

**结论**：当前的导航结构清晰且符合逻辑，特别是将“净值”独立出来，突出了 Moze 对“资产增值”的重视，区别于传统只记流水的 App。

### 1.2 功能完备性审计 (MVP vs. Market)

| 功能维度 | Moze (当前状态) | 主流竞品 (钱迹/随手记/YNAB) | 差距分析 |
| :--- | :--- | :--- | :--- |
| **基础记账** | ✅ 支持收/支/转，二级分类，标签，多账户。 | ✅ 极速记账，模板，自动识别。 | 缺失：**周期性交易**（订阅制管理）、**模板记账**、**退款处理**。 |
| **资产管理** | ✅ 账户余额，净值快照，简单趋势。 | ✅ 账户对账，多币种，负债管理。 | 缺失：**多币种汇率换算**、**信用账户还款关联**、**负债/分期管理**。 |
| **预算管理** | ⚠️ 仅月度总预算。 | ✅ 分类预算，滚动预算，项目预算。 | 缺失：**分类预算**（如限制餐饮支出）、**预算结余结转**。 |
| **报表分析** | ✅ 收支趋势，分类饼图，年度视图。 | ✅ 资产负债表，现金流量表。 | 缺失：**资产负债表**（标准会计视角）、**成员消费对比**。 |
| **数据开放** | ❌ 暂无。 | ✅ Excel/CSV 导入导出。 | **致命缺失**：无法导入历史数据，用户迁移成本极高；无法导出备份，缺乏安全感。 |
| **多端同步** | ⚠️ 仅 Web/PWA。 | ✅ App + Web + Watch + 小程序。 | 缺失：**原生 App** 体验（离线记账、通知推送、桌面小组件）。 |

## 2. 市场竞争分析 (SWOT)

### 优势 (Strengths)
1.  **现代化 UI/UX**：基于 shadcn-vue 与 Tailwind，视觉清爽，无广告，交互流畅，符合极简主义审美。
2.  **资产增值视角**：不仅关注收支流水，更重视“净值”变化，符合中高阶理财用户需求。
3.  **大屏 Web 体验**：弥补了移动端记账 App 在 PC/Pad 上体验差的短板，适合深度复盘。
4.  **数据掌控**：基于 Supabase，理论上支持自部署，数据隐私有保障。
5.  **无理财推荐**：纯净工具，专注财务管理本身。

### 劣势 (Weaknesses)
1.  **数据迁移门槛**：不支持导入其他 App 数据，冷启动困难。
2.  **移动端缺失**：虽然 PWA 可用，但缺乏原生体验（如快捷指令记账、通知栏记账），碎片化记账不便。
3.  **功能深度不足**：缺乏高级预算、借贷管理、分期付款等复杂场景支持。
4.  **生态封闭**：目前没有 API 开放，无法与自动化工具（如 iOS 快捷指令）联动。

### 机会 (Opportunities)
1.  **AI 辅助记账**：结合 LLM 实现自然语言记账、账单截图识别、财务建议生成。
2.  **开源社区**：若开源，可吸引开发者贡献插件（如自动抓取银行账单）。
3.  **家庭/协作场景**：弥补市场上家庭记账体验不佳的空缺。
4.  **FIRE (Financial Independence, Retire Early) 工具**：深耕净值与被动收入分析，吸引 FIRE 人群。

### 威胁 (Threats)
1.  **竞品免费策略**：主流 App 基础功能免费且强大。
2.  **自动化记账工具**：如 MoneyWiz 等支持银行直连的工具，极大降低记账成本。

## 3. 后续发展规划 (Roadmap)

### 阶段一：补齐短板 (短期 1-3 个月)
目标：降低用户迁移门槛，完善核心记账体验。
1.  **数据导入/导出**：支持标准 CSV 导入（适配钱迹/随手记格式），支持全量数据导出备份。
2.  **周期性交易**：支持设置每月/每周自动记账（如房租、订阅费）。
3.  **PWA 体验优化**：优化移动端触摸交互，支持离线记账（Service Worker）。
4.  **搜索功能**：支持按金额、备注、分类搜索历史交易。

### 阶段二：增强分析 (中期 3-6 个月)
目标：提升数据价值，辅助财务决策。
1.  **高级预算**：支持分类预算（如“餐饮”限额 2000），支持预算超支提醒。
2.  **资产负债表**：生成标准的月度/年度资产负债表，清晰展示财务健康度。
3.  **多币种支持**：基础汇率换算，资产折合本币展示。
4.  **标签管理增强**：支持标签分组、标签报表分析。

### 阶段三：智能化与生态 (长期 6-12 个月)
目标：降低记账成本，拓展使用场景。
1.  **AI 记账助手**：接入 LLM，支持“昨天打车花了 35”自然语言记账。
2.  **API 开放**：提供 REST API，支持 iOS 快捷指令自动记账。
3.  **多用户协作**：支持家庭账本，权限管理（只读/读写）。
4.  **投资组合追踪**：对接股票/基金 API，自动更新持仓净值。

---

## 4. 总结

Moze 是一款**“小而美”且“重资产”**的现代化财务管理工具。其核心竞争力在于**优秀的 Web 体验**与**净值管理理念**。

短期内，应优先解决**数据导入导出**与**移动端 PWA 体验**问题，打通用户流入通道。
中期来看，深耕**报表分析**与**预算控制**，帮助用户不仅“记清楚”，还能“管得住”。
长期来看，利用 **AI** 与 **API** 降低记账阻力，实现自动化财务管理。
