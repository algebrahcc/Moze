# 财富板块 FIRE 功能规划与设计 (Wealth & FIRE Features Roadmap)

## 1. 设计理念 (Philosophy)

当前的“财富”板块仅提供了基础的资产快照记录功能。为了服务于追求 **FIRE (Financial Independence, Retire Early)** 的用户群体，该板块需要从单纯的“记账”转变为“规划与监控”。

核心目标：帮助用户清晰地回答三个问题：
1. **我现在有多少钱？** (净资产与资产配置)
2. **我还需要多少钱才能自由？** (FIRE Number & 进度条)
3. **我还要多久才能达到目标？** (储蓄率与复利预测)

---

## 2. 核心功能模块 (Feature Modules)

### 2.1 资产全景与配置 (Advanced Asset Allocation)

FIRE 追求者非常关注资产的风险分布。

*   **资产分类扩展**:
    *   不仅是“股票”或“现金”，需要支持自定义的大类资产配置：
        *   权益类 (Equities): 国内股票, 美股, 基金/ETF
        *   固收类 (Fixed Income): 债券, 货币基金, 定期存款
        *   现金类 (Cash): 活期, 应急备用金
        *   房地产 (Real Estate): 自住, 投资
        *   另类投资 (Alternatives): 加密货币, 黄金, P2P等
*   **目标配比 vs. 实际配比 (Target vs. Actual)**:
    *   用户设定理想的资产配比（例如：60% 股票 / 40% 债券）。
    *   系统展示当前偏离度，并给出**再平衡建议 (Rebalancing)**（例如：“需卖出 ￥10,000 股票，买入 ￥10,000 债券”）。
*   **净资产透视**:
    *   **总净资产 (Total Net Worth)**
    *   **可投资净资产 (Investable Net Worth)**: 剔除自住房产等非流动资产，这是计算 FIRE 收入的核心基数。

### 2.2 FIRE 仪表盘 (The FIRE Dashboard)

这是财富板块的主页，一目了然地展示“自由进度”。

*   **FIRE 数字 (The FIRE Number)**:
    *   **计算公式**: `年支出 × 25` (基于 4% 法则) 或 `年支出 / 自定义提现率`。
    *   **动态调整**: 允许用户基于“当前年支出”或“预设理想年支出”进行计算。
*   **自由进度条 (Progress Bar)**:
    *   显示 `当前可投资资产 / FIRE Number` 的百分比。
    *   里程碑标记：
        *   **Coast FIRE**: 即使不再存钱，靠复利也能在退休时达标的金额。
        *   **Lean FIRE**: 维持最低生存标准的金额。
        *   **Fat FIRE**: 理想富裕生活的金额。
*   **安全跑道 (Runway)**:
    *   `流动资产 / 月均支出` = 不工作能生存的月数/年数。

### 2.3 核心指标监控 (Key Metrics)

*   **储蓄率 (Savings Rate)**:
    *   FIRE 的核心加速器。
    *   公式: `(收入 - 支出) / 收入`。
    *   展示: 月度储蓄率趋势图，以及年度平均储蓄率。
    *   *数据来源*: 直接关联现有的“报表”模块数据。
*   **投资回报率 (ROI)**:
    *   尝试计算简单的 **TWR (时间加权收益率)** 或 **XIRR**（如果数据允许）。
    *   至少提供：本金投入 vs. 当前市值的对比曲线。

### 2.4 预测与模拟 (Projection & Simulation)

回答“还有多久退休”的问题。

*   **蒙特卡洛模拟 (Monte Carlo Simulation)** (高级功能):
    *   基于历史市场数据，模拟未来 30-50 年的成功概率。
*   **简单复利计算器**:
    *   输入：当前资产、月储蓄额、预期年化收益率、预计通胀率。
    *   输出：预计达到 FIRE 状态的年份/日期。
    *   可视化：资产增长曲线与 FIRE 目标线的交点。

### 2.5 提现策略 (Withdrawal Phase - Future)

针对已经达成 FIRE 的用户。

*   **安全提现率 (Safe Withdrawal Rate, SWR)**:
    *   监控当前的实际提现率是否低于 4% (或设定值)。
*   **现金流桶策略 (Bucket Strategy)**:
    *   规划 1-2 年的现金桶，3-5 年的稳健桶，5 年以上的进取桶。

## 3. 数据流向架构 (Data Architecture)

为了实现上述功能，需要打通现有的各个模块：

1.  **账户 (Accounts) -> 资产 (Assets)**:
    *   现有 `accounts` 表需要增加 `asset_class` (资产类别) 字段。
    *   现有 `asset_snapshots` 表继续用于记录市值，但需要更频繁或自动化的更新机制（例如手动对账时的快捷入口）。

2.  **交易 (Transactions) -> 储蓄率 (Savings Rate)**:
    *   利用现有的收支数据计算储蓄率。
    *   需要准确区分“投资性支出”（不应算作消费）和“消费性支出”。
        *   *建议*: 在 `categories` 中增加 `is_investment` 标记，或者通过 `Account Group` 逻辑（如资金流向投资账户不视为支出）。

3.  **设置 (Settings) -> FIRE 参数**:
    *   用户配置：预期年化收益率、通胀率、SWR (安全提现率)、FIRE 倍数 (25x/30x)。

## 4. 实施路线图 (Implementation Roadmap)

### Phase 1: 基础建设 (Foundation)
*   [x] **数据库更新**: `accounts` 表增加 `asset_class` 字段；新建 `fire_settings` 表或在 `user_settings` 中增加字段。
*   [x] **资产配置页**: 改造现有的 `investments/index.vue`，增加饼图展示资产分布。
*   [x] **净资产趋势优化**: 区分“总净资产”与“可投资净资产”。

### Phase 2: FIRE 核心 (The Core)
*   [x] **FIRE 仪表盘**: 开发新的 Dashboard 组件，展示 FIRE Number 和进度条。
*   [ ] **储蓄率集成**: 从报表模块提取数据，在财富页展示储蓄率趋势。
*   [ ] **简单预测**: 实现“距离退休还有 X 年”的计算逻辑。

### Phase 3: 高级功能 (Advanced)
*   [ ] **多币种支持**: (如果用户持有美股/港股) 自动汇率换算。
*   [ ] **投资收益率分析**: 引入 XIRR 计算（需要更复杂的资金进出记录）。
*   [ ] **退休模拟器**: 交互式的参数调整工具。

---

## 5. UI/UX 建议 (Design Concepts)

*   **色调**: 保持红/绿语义的一致性（收益/增长为红，亏损/支出为绿）。
*   **激励性**: 当用户储蓄率提升或资产突破里程碑时，给予视觉反馈（Confetti 撒花效果等）。
*   **清晰度**: 复杂的金融概念（如 SWR）应提供 Tooltip 解释。
