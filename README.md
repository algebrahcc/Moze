# Moze / 默资记账

个人记账 + 投资账户净值跟踪的桌面应用（Tauri），同步支持 Web 端（Nuxt）。股票账户采用“净值记账法”：不记录持仓明细，只按日期记录每个股票账户的总资产与净入金，自动计算盈亏与趋势。

## 目标

- 日常收支：支出/收入/转账，分类与标签
- 投资账户：多个股票账户的每日总资产、净入金、自动盈亏
- 总览看板：净资产、今日盈亏、资产趋势
- 多端同步：本地优先 + 云端同步（Supabase）

## 技术栈

- 应用框架：Nuxt 4（Vue 3 + TypeScript）
- 桌面端：Tauri 2
- UI：shadcn-vue（通过 shadcn-nuxt）+ Tailwind CSS
- 状态/工具：@vueuse/core
- 数据与鉴权：Supabase（PostgreSQL + Auth + RLS）
- 图表（计划）：ECharts（资产曲线、月度收支等）
- 图标：Iconify（通过 Iconify JSON 图标集）

## 数据模型（MVP）

核心按“账户”统一建模，股票账户与现金账户同属账户类型；股票账户的每日净值使用快照表记录。

- accounts
  - type: cash | credit | stock
  - currency: CNY | USD | HKD ...
- transactions（日常收支/转账）
- asset_snapshots（投资账户每日快照）
  - account_id（关联到 type=stock 的账户）
  - date（日期）
  - total_value（当日总资产）
  - net_deposit（当日净入金，可为负）
  - daily_pnl（可由 total_value/昨日 total_value/net_deposit 推导）

## 功能规划

### MVP（先做可用）

- 账户
  - 新建/编辑/归档账户（现金/信用/股票）
- 日常记账
  - 支出/收入/转账
  - 分类管理（支持二级分类）
- 股票账户净值
  - 按日期录入 total_value 与 net_deposit
  - 自动计算当日盈亏与累计盈亏
  - 列表 + 近 30/90 天趋势
- 看板
  - 净资产、今日盈亏、资产趋势

### V2（增强分析）

- 报表：月度/年度收支、分类占比、预算
- 投资分析：多账户对比、资金加权收益率、区间收益
- 数据导入：从 CSV/Excel 导入历史流水与净值

### V3（体验提升）

- 快捷键快速记账（桌面端）
- 多端协作（家庭账本，可选）
- 智能录入（可选）

### V4（小程序端，最后实现）

以“快速录入 + 轻量查看”为目标，小程序端作为独立客户端接入同一套 Supabase 后端与数据模型：

- 快速记账（支出/收入/转账）
- 股票账户净值录入（按日期 total_value + net_deposit）
- 今日/本周概览（净资产、今日盈亏、近 7/30 天趋势）

## 环境变量（Supabase）

在项目根目录创建 `.env`（不要提交到仓库），并配置：

```bash
SUPABASE_URL="https://<project-ref>.supabase.co"
SUPABASE_KEY="<anon/public key>"
```

## 开发运行

安装依赖：

```bash
npm install
```

启动开发服务器（默认 http://localhost:3000）：

```bash

```

## 桌面端（Tauri）

Tauri 配置位于 `src-tauri/`，默认使用：

- devServer：`http://localhost:3000`
- build 输出：`dist`（用于 production 打包）

常见流程：

1) 启动 Nuxt dev server：`npm run dev`
2) 启动 Tauri（需要系统已安装 Rust 工具链与平台依赖）：`npx tauri dev`

## 约定

- 不在代码与文档中写入任何密钥/Token；只通过环境变量注入
- 股票账户使用净值记账法：不追踪持仓、只追踪账户总资产与净入金

## 视觉设计方案（确定方向）

### 方向选择

**极简 · 静默交互 · 连接消费与投资**  
品牌气质为“默资”：低噪、可信、克制，不以视觉刺激驱动，而以秩序与清晰建立信任。

### 用户与场景

- 轻度记账用户：移动端高频录入，期望简洁与低打扰  
- 投资账户用户：关注净值录入与趋势回看  
- 两类用户都需要：快速、可靠、可持续的记录体验

### 视觉与体验原则

- 视觉降噪：减少不必要的装饰，强调数据层级  
- 静默反馈：重要状态可见，但不干扰操作  
- 可信克制：节制的对比与留白塑造“金融可信”  
- 一体化：消费与投资在同一秩序内被理解

## 设计系统（Design System）

### 色彩系统

**浅色模式**

- 背景：hsl(0 0% 100%)  
- 前景：hsl(240 10% 3.9%)  
- 主色：hsl(240 5.9% 10%)  
- 次级：hsl(240 4.8% 95.9%)  
- 语义色：  
  - 成功：#16A34A  
  - 警告：#F59E0B  
  - 错误：hsl(0 84.2% 60.2%)  
  - 信息：#2563EB  

**深色模式**

- 背景：hsl(240 10% 3.9%)  
- 前景：hsl(0 0% 98%)  
- 主色：hsl(0 0% 98%)  
- 次级：hsl(240 3.7% 15.3%)  
- 语义色：保持同色系，提升明度以保证对比

### 字体规范

- 中文：思源黑体（正文）+ 思源宋体（标题）  
- 英文：IBM Plex Sans（正文）+ Fraunces（标题）  
- 字号体系：12 / 14 / 16 / 18 / 24 / 32 / 40  
- 行高：正文 1.6，标题 1.2

### 组件规范

- Button：主/次/描边/禁用/加载  
- Input：默认/聚焦/错误/禁用  
- Card：分区清晰、弱阴影、轻边界  
- Alert：仅错误/成功/提示三类  
- Tabs：紧凑型，强调切换效率  

### 图标体系

- 线性图标为主（1.5px）  
- 保持简洁、可识别、低视觉侵入  

### 间距与栅格

- 4pt 基准：4/8/12/16/24/32/40/56  
- 页面容器最大宽度：1200-1400  
- 移动端最小边距：16px

## 登录页方案（已确定并落地）

### 方案名称：Quiet Ledger

**目标**：极简可信，降低登录焦虑  
**结构**：左侧品牌叙事，右侧静默卡片  
**移动端**：单列堆叠，信息保留但缩减  

### 主要信息结构

1. 品牌标识 + 一句话定位  
2. “连接消费与投资”的价值表达  
3. 登录区域：密码 / 魔法链接 / 注册切换  
4. 操作反馈：错误信息、验证提示

### 交互说明

- 默认密码登录  
- 登录成功自动进入看板  
- 注册成功提示邮件验证  
- 状态反馈低干扰但清晰
